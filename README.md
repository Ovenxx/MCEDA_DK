Multi-Consensus Based Estimation of Distribution Algorithm with Domain Knowledge for Multi-Objective Green Vehicle Routing

The green vehicle routing problem with time window (GVRPTW) is concerned with assigning a fleet of vehicles to a sequence of delivery tasks with specific time windows. The objective is to minimize economic and environmental costs. Existing studies on this problem typically only consider a single type of customer demand and assume constant vehicle speeds. In practice, however, customer demands are diverse, and vehicle speeds vary spatiotemporally. Accordingly, this paper formulates a multi-objective time-space-dependent green VRP with pickup-delivery and time window (MO-TSD-GVRPPDTW), which simultaneously incorporates pickup demands, delivery demands, and time-space dependences. To solve this problem, a multi-consensus-based estimation of distribution algorithm embedded with domain knowledge (MCEDA-DK) is proposed. It comprises three key components: 1) a domain knowledge embedding (DKE) method; 2) a multi-consensus evolution (MCE) strategy; and 3) a multi-scale neighborhood search (MNS) strategy. The DKE method constructs a time-space compatibility matrix to prune the search space, improving both initialization quality and evolution efficiency. The MCE strategy iteratively updates the population’s consensus on multiple conflicting objectives to indicate diversified non-dominated combinations, guiding subsequent searches. The MNS strategy stimulates the population to search both large- and small-scale neighborhoods, enabling persistent exploration of better local regions and identification of local non-dominated solutions, which effectively balances diversified and intensified searches.

The file data.mat contains the spatial parameters used in the experiments. 
The core implementation of the proposed MCEDA-DK algorithm is provided in MCEDA_DK.m. 
Given a problem instance, this algorithm returns the corresponding set of non-dominated routes and the approximated Pareto front in the objective space. 
For performance analysis and comparison, run Experiment_script_100.m, Experiment_script_200.m, and Experiment_script_400.m, then execute HV_computation.m to compute the hypervolume (HV) metric.
